# M4: Frontend (TanStack Start)

## Goal

Build all UI pages for TaskForge features using TanStack Start (SSR + SPA hybrid), tRPC client, and the M3.5 design system.

## Pages

### Auth (may already exist in boilerplate)

- `/auth/login` — login form
- `/auth/register` — register form

### Projects

- `/` → redirect to `/projects`
- `/projects` — project list + create button
- `/projects/new` — create project form (or modal)
- `/projects/$projectId` — project detail (tasks list, tabs: Tasks | Members)
- `/projects/$projectId/settings` — project settings (name, archive, delete, transfer ownership)
- `/projects/$projectId/members` — member management (invite, role change, remove)

### Tasks

- Task create/edit via modal or slide-over on project detail page
- Task detail at `/projects/$projectId/tasks/$taskId` (optional, can be modal)

### Billing

- `/settings/billing` — current plan, upgrade/downgrade button, cancel

### Profile

- `/settings/profile` — user name/email (already in boilerplate)

## tRPC Client Integration

Use the existing tRPC client setup. Call procedures from TanStack Query loaders.

- M4-T4: Project members page (invite, role change, remove, transfer ownership)
- M4-T5: Task detail (update, assign, transition) — modal or dedicated page
- M4-T6: Billing page (current plan, checkout redirect)
